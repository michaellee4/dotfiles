# Dotfiles

This repo manages macOS dotfiles using symlinks. All configs live here and are symlinked to their expected locations via `install.sh`.

## Adding a new config

1. Copy the config file or directory into this repo:
   ```sh
   cp -r ~/.config/sometool ~/dotfiles/sometool
   ```

2. Add a symlink entry to `install.sh`:
   ```sh
   symlink sometool "$HOME/.config/sometool"
   ```

3. Remove the original so the symlink can be created:
   ```sh
   rm -rf ~/.config/sometool
   ```

4. Run the install script to verify it works:
   ```sh
   ./install.sh
   ```

5. Add the tool to the README table with a description and link to its GitHub.

6. Commit and push.

## Notes

- `install.sh` is safely re-runnable — it uses `ln -sf` for files and warns if a real directory exists at the target path
- Whole directories are symlinked (not individual files) so new files added to a config dir are automatically tracked
- `nvim/lazy-lock.json` is excluded via `.gitignore` as it is auto-generated by lazy.nvim
